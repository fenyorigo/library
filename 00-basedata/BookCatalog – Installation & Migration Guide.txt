BookCatalog – Installation & Migration Guide (macOS → Linux)

Audience:
	•	future-you
	•	a technically competent admin
	•	not a developer deep-dive

Non-goals:
	•	no CI/CD
	•	no containerization (unless you later want it)
	•	no HA / clustering

⸻

1. Purpose & scope
	•	Describe requirements, assumptions, and steps
	•	Cover:
	•	fresh install
	•	migration from existing instance
	•	Concrete reference platforms:
	•	Source: macOS Tahoe (Apple Silicon)
	•	Target: Fedora 43 Server + KDE Plasma

⸻

2. Reference setup – macOS Tahoe (source system)

This is where you document exactly what you are running today.

2.1 Operating system
	•	macOS Tahoe
	•	Apple Silicon (ARM64 / AArch64)

2.2 Database
	•	Oracle MySQL Server (genuine AARCH64 build)
	•	installed via official .dmg / pkg
	•	service managed by launchd
	•	Default charset/collation
	•	Port, socket location

2.3 PHP
	•	Installed via Homebrew
	•	PHP version
	•	Enabled extensions (important!):
	•	mysqli / pdo_mysql
	•	zip
	•	json
	•	mbstring
	•	openssl
	•	php.ini location

2.4 Web server
	•	Apache (brew or system — whichever you use)
	•	DocumentRoot
	•	VirtualHost config
	•	PHP handler (mod_php or php-fpm)

2.5 Tooling
	•	Homebrew
	•	Node.js (for frontend build)
	•	mysqldump location
	•	Backup scripts in ~/bin

2.6 File layout (authoritative)
	•	Project root
	•	public/uploads
	•	00-basedata
	•	Backup locations

This section becomes the baseline truth.

3. Target platform – Fedora 43 with KDE Plasma

This mirrors section 2, but for Fedora.

3.1 Operating system
	•	Fedora 43 Server
	•	KDE Plasma GUI
	•	SELinux mode (important!)

3.2 Database (MySQL / MariaDB decision)

Here we explicitly decide and document:
	•	MySQL (Oracle community) or
	•	MariaDB (Fedora default)

Include:
	•	package names
	•	service management (systemctl)
	•	data directory
	•	socket vs TCP
	•	charset/collation alignment with macOS

3.3 PHP
	•	Fedora packages
	•	Version parity with macOS
	•	Required extensions (same list as above)
	•	php.ini paths (CLI vs Apache)

3.4 Apache / httpd
	•	httpd packages
	•	VirtualHost example
	•	PHP integration
	•	File permissions model
	•	SELinux contexts for public/uploads

This is where Fedora-specific gotchas live.

4. Migration procedure (macOS → Fedora)

Step-by-step, no branching logic.

4.1 Prepare source system
	•	Create DB dump (backup_bpbooks.sh)
	•	Create project snapshot
	•	Verify backups

4.2 Prepare target system
	•	Install OS packages
	•	Enable services
	•	Create database and user

4.3 Restore data
	•	Import SQL dump
	•	Restore uploads
	•	Verify permissions

4.4 Deploy application
	•	Copy project files
	•	Configure vhost
	•	Configure PHP
	•	Build frontend

4.5 Validation checklist
	•	Login works
	•	Reader vs admin roles
	•	Covers visible
	•	Backup works on target system

⸻

5. Known differences & pitfalls

This is where the document really pays off later:
	•	ARM macOS vs x86_64 Fedora
	•	File permissions vs SELinux
	•	MySQL socket paths differ
	•	PHP extension names differ
	•	Apache config layout differs
